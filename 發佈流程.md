VS Code 擴充套件完整發布流程
0. 先準備好擴充套件專案

你的專案至少要有這些：

package.json

編譯後的輸出檔，例如 out/extension.js

README.md

如果還沒建立專案，官方建議可用 yo code 產生基本骨架。

1. 安裝發布工具 vsce

先安裝官方常用的發佈工具：

npm install -g @vscode/vsce

vsce 用來做兩件事：

打包成 .vsix

發布到 VS Code Marketplace

2. 建立 Publisher

到 Marketplace 的 Publisher 管理頁：

https://marketplace.visualstudio.com/manage/publishers/

如果還沒有 publisher，就按 Create publisher。
每個 extension 都必須有一個 publisher，而且這個值要寫進 package.json。

建立時你會填：

Name：顯示名稱

ID：真正的 publisher ID，之後會用在 package.json

例如：

"publisher": "ericlifetw"
3. 建立 Personal Access Token（PAT）

你要建立一組 PAT，讓 vsce 可以代表你發佈。

重點是：

建 PAT 的 Microsoft 帳號

要和建立 publisher 的 Microsoft 帳號相同

這點官方有特別提醒。

通常在 Azure DevOps 的使用者設定頁建立 PAT。你之前已經成功建立並驗證了，所以這一步你已完成。

4. 用 PAT 登入 publisher

在專案目錄執行：

vsce login ericlifetw

然後貼上 PAT。

成功會看到類似：

The Personal Access Token verification succeeded for the publisher 'ericlifetw'.

這代表你已經把本機 vsce 和 Marketplace publisher 綁定成功。

5. 修改 package.json

這一步最重要，至少要確認下面幾個欄位：

{
  "name": "commit-message-anyai2helper",
  "displayName": "AnyAI Commit Message Helper",
  "publisher": "ericlifetw",
  "version": "0.0.1",
  "engines": {
    "vscode": "^1.80.0"
  },
  "main": "./out/extension.js"
}
這裡要注意 3 件事
publisher

要和你剛建立的 publisher 一樣：

"publisher": "ericlifetw"
name

這是技術名稱，不能和 Marketplace 既有 extension 衝突。
你前面就遇到過 commit-message-helper 已被使用的情況。

displayName

這是顯示名稱，也可能被別人占用。
你前面也遇到過 display name 重複，所以要改成獨特名稱。

這些都會影響是否能成功發布。

6. 建議補齊的檔案

雖然有些不是硬性必需，但很建議上架前補好：

README.md

LICENSE

CHANGELOG.md

repository 欄位

官方發佈文件有列出這些是建議一起提供的內容。

例如 package.json：

"repository": {
  "type": "git",
  "url": "https://github.com/你的帳號/你的repo.git"
}
7. 不要把不必要檔案一起打包

你之前的 VSIX 裡包含了這些：

.claude/

.vscode/

infor.txt

infor2.txt

圖片檔

這些通常不建議一起發布。

你可以在專案根目錄新增 .vscodeignore：

.vscode/**
.claude/**
*.txt
src/**
tsconfig.json

這樣 vsce 打包時就不會把這些一起帶上去。

8. 先本機打包測試

先執行：

vsce package

這會產生一個 .vsix 檔，讓你可以先自己安裝測試。

自己安裝方式：

code --install-extension your-extension-name.vsix

這一步很適合拿來先驗證：

指令有沒有註冊成功

extension 有沒有正常啟動

UI 有沒有跑起來

9. 正式發布

確認都沒問題後，直接發布：

vsce publish

如果版本號已經存在，改用：

vsce publish patch

它會自動把版本從例如：

0.0.1 → 0.0.2

再幫你發佈。

10. 發布成功後會得到什麼

成功後會看到類似：

Published ericlifetw.commit-message-anyai2helper v0.0.1

也會拿到 Marketplace 連結，例如你的：

https://marketplace.visualstudio.com/items?itemName=ericlifetw.commit-message-anyai2helper

通常 Marketplace 頁面會先可開啟，但 VS Code 搜尋結果可能要過幾分鐘才會出現。Marketplace 是 VS Code 安裝與搜尋 extension 的官方來源。

你這次實際遇到的常見錯誤整理
錯誤 1：PAT 驗證失敗

原因通常是：

publisher 還沒建立

PAT 所屬帳號和 publisher 帳號不同

你登入了錯的 publisher 名稱

錯誤 2：name 已存在

例如：

The extension 'commit-message-helper' already exists

代表 package.json 的 name 和別人重複。

錯誤 3：displayName 已存在

例如：

This extension display name is taken

代表 displayName 和別人重複。

錯誤 4：VSIX 打包太髒

把 .vscode、.claude、測試檔、說明截圖全包進去。
這雖然不一定會擋住發佈，但會讓 Marketplace 包變得很雜。

建議的實戰發布順序

照這個走最穩：

# 1 安裝工具
npm install -g @vscode/vsce

# 2 登入 publisher
vsce login ericlifetw

# 3 先打包測試
vsce package

# 4 正式發布
vsce publish

若版本衝突：

vsce publish patch
上架後建議立刻補強的 5 件事
1. 補 icon

在 package.json 加：

"icon": "icon.png"
2. 補 keywords

讓搜尋更容易命中：

"keywords": [
  "git",
  "commit",
  "ai",
  "commit message"
]
3. 補 README 截圖

可以讓 Marketplace 頁面比較完整。

4. 改掉 activationEvents: ["*"]

官方提醒 * 會影響效能，通常不建議。

5. 注意 extension 的安全性

VS Code 官方提醒，extension 執行時能讀寫檔案、發出網路請求、執行外部程式，所以不要把敏感資訊硬寫進 extension。
